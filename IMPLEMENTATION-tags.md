# Tags Implementation in AkashaRender

This document describes the implementation of document tags in AkashaRender, including data structures, APIs, and CLI commands.

## Overview

Document tags are metadata defined in the frontmatter of each document. Tags help categorize content and enable features like tag index pages and content filtering.

### Frontmatter Example

```yaml
---
title: My Article
tags:
    - JavaScript
    - Tutorial
    - Web Development
---
```

Tags can also be specified as a comma-separated string:

```yaml
tags: JavaScript, Tutorial, Web Development
```

## Data Structures

### TagDescription

Defined in `lib/types.ts`:

```typescript
interface TagDescription {
    tagName: string;
    description: string;
}
```

Tag descriptions provide human-readable explanations for tags, typically displayed on tag index pages generated by `@akashacms/plugins-tagged-content`.

### Database Tables

#### TAGGLUE Table

Maps documents to their tags. Defined in `lib/cache/sql/create-table-tagglue.sql`:

```sql
CREATE TABLE IF NOT EXISTS
TAGGLUE (
    docvpath STRING,
    tagName STRING
);
```

- `docvpath`: The virtual path of the document
- `tagName`: A tag name used by that document

Multiple rows exist for documents with multiple tags.

#### TAGDESCRIPTION Table

Stores tag descriptions. Defined in `lib/cache/sql/create-table-tag-description.sql`:

```sql
CREATE TABLE IF NOT EXISTS
TAGDESCRIPTION (
    tagName STRING UNIQUE,
    description STRING
);
```

## Configuration

### Adding Tag Descriptions

In your configuration file (e.g., `config.mjs`):

```javascript
config.addTagDescriptions([
    {
        tagName: 'JavaScript',
        description: 'Articles about JavaScript programming'
    },
    {
        tagName: 'Tutorial',
        description: 'Step-by-step learning guides'
    }
]);
```

## API Reference

### TagGlue Class (`lib/cache/tag-glue.ts`)

#### `tags(): Promise<string[]>`

Returns a list of all unique tag names used across all documents.

#### `pathsForTag(tagName: string | string[]): Promise<string[]>`

Returns document vpaths that use the specified tag(s).

#### `findSimilarTags(threshold?: number): Promise<SimilarTagGroup[]>`

Finds groups of similar tags based on:
- Case-insensitive matching (e.g., `JavaScript` vs `javascript`)
- Plural/singular variants (e.g., `tag` vs `tags`)
- Levenshtein distance within threshold (default: 2)

Returns:
```typescript
interface SimilarTagGroup {
    tags: string[];
    reasons: ('case-insensitive' | 'plural-singular' | 'levenshtein')[];
    documentsByTag: Record<string, string[]>;
}
```

#### `tagsWithoutDescriptions(): Promise<TagWithoutDescription[]>`

Finds tags used in documents that have no entry in TAGDESCRIPTION.

Returns:
```typescript
interface TagWithoutDescription {
    tagName: string;
    documents: string[];
}
```

### TagDescriptions Class (`lib/cache/tag-glue.ts`)

#### `addDesc(tag: string, description: string): Promise<void>`

Adds or updates a tag description.

#### `getDesc(tag: string): Promise<string | undefined>`

Retrieves the description for a tag, or undefined if not found.

#### `unusedTagDescriptions(): Promise<string[]>`

Finds tag descriptions that exist in the database but are not used by any document.

### DocumentsCache Wrapper Methods (`lib/cache/cache-sqlite.ts`)

The DocumentsCache class provides wrapper methods for tag operations:

- `tags()` - Calls `TagGlue.tags()`
- `findSimilarTags(threshold)` - Calls `TagGlue.findSimilarTags()`
- `tagsWithoutDescriptions()` - Calls `TagGlue.tagsWithoutDescriptions()`
- `unusedTagDescriptions()` - Calls `TagDescriptions.unusedTagDescriptions()`
- `documentsWithTag(tags)` - Returns documents matching the specified tag(s)
- `addTagDescription(tag, description)` - Adds a tag description
- `getTagDescription(tag)` - Gets a tag description

### refactorTag Function (`lib/refactor-tags.ts`)

```typescript
async function refactorTag(
    config: Configuration,
    oldTag: string,
    newTag: string,
    options: { dryRun: boolean }
): Promise<RefactorTagResult>
```

Renames a tag across all documents by modifying frontmatter. Supports dry-run mode to preview changes without modifying files.

Returns:
```typescript
interface RefactorTagResult {
    oldTag: string;
    newTag: string;
    dryRun: boolean;
    modifiedDocuments: RefactorDocumentChange[];
    mergedDocuments: RefactorDocumentChange[];  // Documents that already had newTag
    errors: RefactorError[];
}
```

## CLI Commands

All tag-related commands output YAML for easy reading.

### `akasharender tags <configFN>`

Lists all tags used in the site.

```bash
npx akasharender tags config.mjs
```

Output:
```yaml
tags:
    - JavaScript
    - Tutorial
    - Web Development
```

### `akasharender similar-tags <configFN> [--threshold <n>]`

Finds groups of similar tags that may indicate inconsistent tagging.

```bash
npx akasharender similar-tags config.mjs --threshold 2
```

Output:
```yaml
similarTagGroups:
    - tags:
        - JavaScript
        - javascript
      reasons:
        - case-insensitive
      documentsByTag:
        JavaScript:
            - articles/intro.html.md
        javascript:
            - tutorials/basics.html.md
```

### `akasharender tags-without-descriptions <configFN>`

Lists tags that have no description defined.

```bash
npx akasharender tags-without-descriptions config.mjs
```

### `akasharender unused-tag-descriptions <configFN>`

Lists tag descriptions that exist in configuration but are not used by any document.

```bash
npx akasharender unused-tag-descriptions config.mjs
```

### `akasharender docs-with-tag <configFN> <tags...>`

Lists documents that use the specified tag(s).

```bash
npx akasharender docs-with-tag config.mjs JavaScript Tutorial
```

### `akasharender refactor-tag <configFN> <oldTag> <newTag> [--dry-run]`

Renames a tag across all documents.

```bash
# Preview changes
npx akasharender refactor-tag config.mjs javascript JavaScript --dry-run

# Apply changes
npx akasharender refactor-tag config.mjs javascript JavaScript
```

Output:
```yaml
refactorResult:
    oldTag: javascript
    newTag: JavaScript
    dryRun: false
    modifiedDocuments:
        - vpath: tutorials/basics.html.md
          fspath: /path/to/tutorials/basics.html.md
          originalTags:
            - javascript
            - Tutorial
          newTags:
            - JavaScript
            - Tutorial
    mergedDocuments: []
    errors: []
```

## Related Plugins

### @akashacms/plugins-tagged-content

This plugin provides:
- Custom tags for displaying tag clouds and lists
- Generation of tag index pages
- Tag-based content filtering

Repository: https://github.com/akashacms/akashacms-tagged-content

The plugin uses the tag APIs described above to:
1. Generate index pages for each tag
2. Display tag descriptions at the top of index pages
3. List documents categorized by each tag

## Dependencies

- `fastest-levenshtein` - For computing edit distance in similar tag detection
- `pluralize` - For detecting plural/singular tag variants
- `gray-matter` - For parsing and modifying document frontmatter (used in refactoring)
