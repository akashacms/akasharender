## Changes

### New Concept Pages

1. **wiki/concepts/performance-tracing.md** - Complete documentation of the TRACES table system that records rendering stage timing data. Covers data structure (basedir, fpath, renderTo, stage, start, now), recording via data.report(), retrieval via data.print() and data.data4file(), cleanup methods, and integration with render.ts. Sourced from lib/data.ts, lib/render.ts, and lib/sql/data-create-table.sql.

2. **wiki/concepts/key-value-store.md** - Documentation of the SQLite-backed key-value storage API using sq3-kv-data-store package. Explains newSQ3DataStore() factory function, named table isolation pattern, shared database connection approach, and export for plugin use. Medium confidence due to limited codebase documentation of the external package API. Sourced from lib/sqdb.ts and lib/index.ts.

3. **wiki/concepts/event-driven-architecture.md** - Comprehensive coverage of the EventEmitter pattern used throughout the cache system. Documents BaseCache extension of EventEmitter, three main event types ('added', 'ready', 'error'), file watching events ('change', 'add', 'unlink'), listener registration patterns, and integration with watchman.ts and verbose mode. Sourced from lib/cache/cache-sqlite.ts and lib/cache/watchman.ts.
