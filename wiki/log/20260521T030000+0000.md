## Changes

Created six new concept documentation files:

1. **wiki/concepts/database-extensions.md** - Comprehensive documentation of three SQLite extensions (sqlite-regex for pattern matching, sqlite-vec for vector similarity, sqlite-lembed for embeddings), environment variable configuration (AK_LEMBED_MODEL, AK_LEMBED_MODEL_NAME), model registration, and WAL mode. Sourced from lib/sqdb.ts.

2. **wiki/concepts/vector-embeddings.md** - Detailed explanation of 384-dimensional vector representations, vec_documents virtual table using vec0 module, embedding generation via lembed() SQL function, conditional storage based on model configuration, document association via vpath, and update handling. Sourced from lib/cache/sql/create-table-vec-documents.sql, insert-lembed-documents.sql, lib/cache/cache-sqlite.ts, and lib/sqdb.ts.

3. **wiki/concepts/semantic-search.md** - Complete semantic search workflow documentation covering query embedding, MATCH operator for vector similarity, distance ranking with LIMIT 10, result enrichment via JOIN with DOCUMENTS table, CTE pattern usage, and CLI access via docs-semantic command. Sourced from lib/cache/sql/doc-search-semantic.sql, lib/cache/cache-sqlite.ts, and lib/cli.ts.

4. **wiki/concepts/type-validation.md** - Documentation of Joi schema validation pattern, validation function structure returning {error, value} tuples, field type specifications (.optional(), .allow(null), .number(), .any()), type guards for TypeScript narrowing, and validation timing. Sourced from lib/cache/schema.ts and lib/types.ts.

5. **wiki/concepts/site-rendering.md** - Five-stage site rendering workflow (pre-rendering hook, document collection with filtering, concurrent rendering via fastq queue, post-rendering hook, result return), queue processing with configurable concurrency, CLI integration, and error handling for individual documents. Sourced from lib/render.ts and lib/cli.ts.

6. **wiki/concepts/github-pages-publishing.md** - gh-pages-publish CLI command documentation covering extensive configuration options (branch, repo, remote, cname, user credentials, message), gh-pages library integration, force push behavior, authentication requirements, and CNAME file handling. Sourced from lib/cli.ts.
