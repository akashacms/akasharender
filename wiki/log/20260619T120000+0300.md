# 2026-06-19 12:00 - Added Answer Page on ActivityPub and Fediverse Integration Elements

Created a new answer page outlining the elements required for ActivityPub and
Fediverse integration within the AkashaCMS ecosphere, in response to a user
request for such an outline.

The answer clarifies a key correctness point: ActivityPub is a JSON-LD protocol
over HTTP (ActivityStreams 2.0), not an HTML-tag standard. A static site
generator like AkashaRender cannot be a full ActivityPub actor on its own, but
it can produce the static metadata and discovery artifacts that make content
Fediverse-friendly. The page presents a tiered/phased plugin design:

- Tier 1: `<head>` metadata (Open Graph, Twitter Card, and the Mastodon
  `fediverse:creator` convention), grounded in the existing `mahaMetadata`
  injection performed by the Built-in Plugin.
- Tier 2: static discovery endpoints (WebFinger, NodeInfo, host-meta).
- Tier 3: static ActivityPub actor/object JSON-LD plus `rel="alternate"` links.
- Tier 4: live federation, flagged as requiring a server component.

External protocol details were marked NEEDS VERIFICATION since they derive from
web standards rather than AkashaRender source.
