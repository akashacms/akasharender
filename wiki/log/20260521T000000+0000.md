# 2026-05-21 00:00 - Document Nine Core Concepts

This session documented nine previously undocumented concepts from the "To Be Documented" section of wiki/concepts/README.md. The concepts were selected based on their frequency of references in existing summary files and their importance to understanding AkashaRender's architecture.

## Analysis

The wiki/concepts/README.md file listed 30 concepts that were referenced in summaries but not yet documented. After reviewing the existing summaries, the following high-priority concepts were identified for documentation:

**Tag System Concepts** (4 documents):
- Tag System - Core tagging infrastructure
- Tag Similarity Detection - Quality assurance for tags
- Tag Refactoring - Tag maintenance operations
- Levenshtein Distance - Algorithm used by similarity detection

**Built-in Plugin Concepts** (4 documents):
- Built-in Plugin - Automatically-added core functionality
- Link Relativization - URL transformation feature
- Image Resizing - Responsive image generation
- Nunjucks Extensions - Template engine customizations

**Design Patterns** (1 document):
- Dry Run Pattern - Preview-before-execute pattern

## Changes

Created nine new concept documentation files following the wiki guidelines in AGENTS.md:

1. **wiki/concepts/tag-system.md** - Comprehensive documentation of the tag system including database structure, tag management operations, tag descriptions, analysis capabilities, and refactoring support. Sourced from lib/cache/tag-glue.ts, lib/types.ts, and lib/refactor-tags.ts.

2. **wiki/concepts/tag-similarity-detection.md** - Detailed explanation of the three-algorithm similarity detection system (case-insensitive, plural/singular, Levenshtein distance) used to identify tag variants. Sourced from lib/cache/tag-glue.ts and lib/types.ts.

3. **wiki/concepts/levenshtein-distance.md** - Documentation of the string similarity algorithm, how it's used in tag analysis, parameter tuning (threshold), and its limitations with short strings and non-English languages. Sourced from lib/cache/tag-glue.ts.

4. **wiki/concepts/tag-refactoring.md** - Complete workflow documentation for batch tag renaming, including query phase, document processing, merge scenarios, dry-run support, and error handling. Sourced from lib/refactor-tags.ts and lib/types.ts.

5. **wiki/concepts/dry-run-pattern.md** - General documentation of the preview-before-execute pattern, implementation strategy, and when to use it. Sourced from lib/refactor-tags.ts.

6. **wiki/concepts/built-in-plugin.md** - Documentation of the special automatically-added plugin, its registration, template directories, Mahabhuta integration, Nunjucks extensions, link relativization control, and image resizing capabilities. Sourced from lib/built-in.ts and lib/index.ts.

7. **wiki/concepts/link-relativization.md** - Detailed explanation of the three independent relativization options (head links, script links, body links), how URLs are detected and transformed, and the relative path calculation process. Sourced from lib/built-in.ts.

8. **wiki/concepts/image-resizing.md** - Complete documentation of the queued image processing system, including attribute detection, queue addition, deferred execution, and the seven-step processing workflow using sharp library. Sourced from lib/built-in.ts.

9. **wiki/concepts/nunjucks-extensions.md** - Documentation of the three custom Nunjucks template tags (akstylesheets, akheaderjs, akfooterjs), their implementation structure, context access, and HTML generation. Sourced from lib/built-in.ts.
