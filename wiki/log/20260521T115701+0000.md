# 2026-05-21 11:57 - Removed watch mode documentation from wiki

This log documents the removal of watch mode references from the wiki after the watch mode functionality was removed from the AkashaRender codebase (lib/cli.ts and lib/cache/watchman.ts were removed).

## Rationale

The watch mode functionality in AkashaRender was based on the old Chokidar-based cache system and would fail if executed. The watch command and watchman module were removed from the codebase, requiring corresponding documentation cleanup in the wiki to maintain accuracy.

## Changes Made

### Files Deleted

1. **wiki/concepts/file-watching.md** - Deleted entire concept document (was entirely about watch mode functionality)
2. **wiki/summaries/lib/cache/watchman.ts.md** - Deleted summary document for the removed watchman module

### Files Modified

3. **wiki/concepts/command-line-interface.md**
   - Removed "watching for changes" from CLI definition
   - Removed "Development Commands" section containing watch command
   - Removed "Development: Use watch during development" from When To Use section
   - Removed "Watch Mode" risk from Risks & Pitfalls section
   - Removed link to File Watching concept from Related Pages

4. **wiki/concepts/event-driven-architecture.md**
   - Removed "change, add, unlink (File Watching)" event documentation
   - Updated error event description to remove "or watching" reference

5. **wiki/concepts/rendering-pipeline.md**
   - Removed "Incremental Rendering (in watch mode)" section
   - Removed "Per-File Events (during file watching)" lifecycle hooks section
   - Removed "Incremental Builds: Re-render changed files in watch mode" example
   - Removed "Incremental Rendering Correctness" section about cache invalidation in watch mode

6. **wiki/concepts/file-caching.md**
   - Removed watch mode event types: 'add', 'change', 'unlink'
   - Kept only initial scan events: 'added', 'ready', 'error'

7. **wiki/concepts/concurrent-rendering.md**
   - Removed "Layout Change Re-rendering" section about watch mode re-rendering
   - Removed note about watch mode using concurrency of 10

8. **wiki/summaries/lib/cli.ts.md**
   - Removed `watch <configFN>` command from Core Rendering Commands list

9. **wiki/log/20260521T020000+0000.md**
   - Removed reference to file-watching.md concept document from the log entry

## Impact

- All references to watch mode functionality have been removed from the wiki
- Documentation now accurately reflects the current codebase which has no watch functionality
- Users will not be misled by documentation describing non-existent features
- Wiki remains consistent with the actual AkashaRender implementation
