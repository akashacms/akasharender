# 2026-06-17 15:16 - Added promised.node.sqlite Migration Architecture Document

Created an architecture document capturing the plan to migrate AkashaRender's
SQLite stack from `sqlite3` + `promised-sqlite3` to the standalone
`promised.node.sqlite` package built on Node.js's `node:sqlite` module.

This was produced in response to a user request to plan the migration before
making any code changes. The user noted a previous in-tree attempt
(documented in the repository's `MIGRATION-*.md` files) that embedded the
wrapper as `lib/async-node-sqlite.ts`, and explained the decision to instead
extract that wrapper into the independent `promised.node.sqlite` package
(located at `../promised.node.sqlite`, GitHub `robogeek/promised.node.sqlite`,
not yet published to npmjs).

## Analysis

- Identified every source file importing `sqlite3` / `promised-sqlite3`:
  `lib/sqdb.ts`, `lib/data.ts`, `lib/cache/cache-sqlite.ts`,
  `lib/cache/tag-glue.ts`, `lib/cache/schema.ts`. `lib/index.ts` re-exports
  `newSQ3DataStore` but does not import the packages directly.
- Reviewed the `promised.node.sqlite` package (`README.md`, `SPEC.md`,
  `src/index.ts`, `package.json`) and confirmed it exposes a superset of the
  `promised-sqlite3` API (`open`, `run`, `get`, `all`, `each`, `close`,
  `inner`) with compatible parameter binding, zero runtime dependencies, and a
  Node.js 24+ requirement already met by AkashaRender.
- Documented the behavioral differences requiring code changes: opening with
  `allowExtension: true`, removal of the `DatabaseSync` `.on('error')`
  listener, incompatibility of `sqlite3-query-log` profiling, the
  `sq3-kv-data-store` dependency on a raw driver handle, and the strict
  `undefined`/boolean/BigInt binding behavior of `node:sqlite`.
- Flagged decision points: how to reference the unpublished package in
  `package.json`, how to resolve `sq3-kv-data-store`, and whether to remove or
  re-engineer query profiling.
- No source code outside `wiki/` was modified, per the user's instruction and
  the wiki rules.
