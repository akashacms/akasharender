# 2026-05-21 06:17 - Documented Five Rendering System Concepts

This session continued the concept documentation work by creating comprehensive documentation for five rendering-related concepts. These concepts build on the core infrastructure documented in the previous session and focus on how AkashaRender processes documents through its rendering pipeline.

## Analysis

1. **DOM Manipulation** - How HTML is programmatically modified after rendering
2. **Rendering Pipeline** - The complete workflow for transforming a project into a website
3. **Layout Templates** - How page structure is separated from content

## Changes

Created five new concept documentation files following the established wiki page format.

### DOM Manipulation (wiki/concepts/dom-manipulation.md)

- Documented DOM manipulation using Cheerio as the server-side jQuery-like library
- Explained the workflow: parse HTML → select elements → modify → serialize
- Covered common DOM operations: querying, reading/modifying attributes, classes, content, creating/inserting/removing elements, and traversing
- Detailed the five mahafunc types for different manipulation patterns: CustomElement, ElementTweaker, Munger, PageProcessor
- Explained the critical dirty flag requirement for tracking DOM changes
- Documented Cheerio configuration options
- Identified risks including forgetting the dirty flag, inefficient selectors, repeated queries, and Cheerio vs. browser differences

### Rendering Pipeline (wiki/concepts/rendering-pipeline.md)

- Documented the complete site-wide rendering workflow
- Explained the pipeline stages: initialization, discovery, concurrent rendering, individual document processing, result aggregation
- Covered the use of fastq for promise-based concurrent queue processing
- Detailed the RenderingResults tracking structure with performance metrics
- Explained special rendering paths for CSS files and asset copying
- Covered performance optimization through concurrent processing, file caching, and query result caching
- Documented lifecycle hooks for plugins
- Identified risks including excessive/insufficient concurrency, cache readiness, missing error handling, and performance bottlenecks

### Layout Templates (wiki/concepts/layout-templates.md)

- Documented layout templates as reusable page structure separating content from design
- Explained layout application in Stage 2 of three-stage rendering
- Covered layout specification via frontmatter and layout template structure
- Detailed variable substitution in different template engines (EJS, Nunjucks, Handlebars)
- Explained layout discovery through stacked directories
- Documented nested layouts for hierarchical structure
- Covered layout metadata merging and cache schema
- Explained layout change re-rendering in watch mode
- Identified risks including forgetting content variable, escaping issues, circular references, and metadata override confusion

### Concurrent Rendering (wiki/concepts/concurrent-rendering.md)

- Documented concurrent rendering using fastq for parallel processing
- Explained concurrency configuration via config.concurrency (default: 3)
- Covered site-wide rendering concurrency and worker function execution
- Detailed layout change re-rendering with higher concurrency (10)
- Explained performance benefits and memory implications
- Covered resource contention for SQLite database, filesystem, CPU, and I/O
- Documented when to increase/decrease concurrency based on system resources and rendering characteristics
- Identified risks including excessive concurrency causing memory exhaustion, too little concurrency underutilizing resources, and non-deterministic errors

### Frontmatter Parsing (wiki/concepts/frontmatter-parsing.md)

- Documented frontmatter parsing using the gray-matter library
- Explained frontmatter structure: YAML metadata between --- delimiters, followed by content
- Covered parsing API: matter(string), parsed.data, parsed.content
- Explained modifying frontmatter with matter.stringify()
- Detailed how frontmatter flows through rendering stages
- Listed common frontmatter fields: layout, title, author, tags, custom metadata
- Documented YAML data types: strings, numbers, booleans, arrays, objects, null
- Explained metadata inheritance from global config, directory metadata, layout, and document
- Covered tag refactoring as an example of frontmatter modification
- Identified risks including YAML syntax errors, indentation errors, missing delimiters, and data type loss
