# 2026-06-01 11:33 - Added resolveVpath Function to Centralize Vpath Resolution

This log entry documents the implementation of a centralized `resolveVpath()` function to address GitHub issue #51, which identified code duplication and inconsistency in computing vpaths from relative paths across the AkashaRender codebase.

## Background

Issue #51 reported that throughout the AkashaRender codebase and its plugins, there was duplicated code for computing a vpath when given a relative path reference from a document. The code patterns were repeated with slight variations, leading to:

1. **Code duplication**: The same logic implemented multiple times
2. **Inconsistent implementations**: Some used `startsWith('/')` while others used `path.isAbsolute()`
3. **Bug potential**: Different implementations could behave differently
4. **Maintenance burden**: Fixing bugs required changes in multiple locations

The issue specifically cited two examples:
- `ShowContent` class in `lib/built-in.ts` using `href.startsWith('/')`
- Code in `@akashacms/diagrams-maker` using `path.isAbsolute()`

## Changes

### 1. Created `resolveVpath()` Function

**File:** `lib/index.ts` (lines 458-497)

Created a new exported function:

```typescript
export function resolveVpath(baseVpath: string, relativePath: string): string
```

**Features:**
- Takes a base vpath and a relative path
- Returns an absolute vpath in the virtual filesystem
- Uses `path.isAbsolute()` instead of `startsWith('/')` (bug fix)
- Normalizes paths to clean up `..` and `.` segments
- Includes input validation with descriptive error messages
- Comprehensive JSDoc documentation with examples

**Implementation:**
```typescript
export function resolveVpath(baseVpath: string, relativePath: string): string {
    if (!baseVpath || typeof baseVpath !== 'string') {
        throw new Error(`resolveVpath: baseVpath must be a non-empty string, got ${typeof baseVpath}`);
    }
    if (!relativePath || typeof relativePath !== 'string') {
        throw new Error(`resolveVpath: relativePath must be a non-empty string, got ${typeof relativePath}`);
    }

    // If the path is already absolute, return it normalized
    if (path.isAbsolute(relativePath)) {
        return path.normalize(relativePath);
    }

    // Get the directory of the base vpath
    const dir = path.dirname(baseVpath);
    
    // Join with '/' prefix to ensure we get an absolute vpath
    // and normalize to clean up any .. or . segments
    return path.normalize(path.join('/', dir, relativePath));
}
```
