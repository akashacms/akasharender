# 2026-06-27 19:10 - Add oEmbed provider implementation guide

Created an implementation-design document that turns the oEmbed provider
architecture into a concrete coding plan for `@akashacms/plugins-base`. The
plan was derived after reading the document-cache schema and populate logic
(`lib/cache/schema.ts`, `lib/cache/cache-sqlite.ts`) to confirm the exact
metadata fields available (`renderPath`, `metadata.title`,
`metadata.rendered_url`, `docMetadata.author`), and the existing
`ak_linkreltag.html.ejs` partial which only emits `rel`/`href` and therefore
must be extended for oEmbed's `type`/`title` attributes.

## Changes

- Created `wiki/implementation/oembed-provider.md` (type: implementation)

  Backlinks sections. Includes the `generateOEmbed` flag API, a shared

  `ak_oembed_links.html.njk` partial, per-page JSON/XML generation folded

  five-phase implementation order, and gotchas.
