# 2026-06-27 19:01 - Add oEmbed provider architecture document

Created a new architecture document describing how to implement an oEmbed
provider for AkashaCMS, in response to GitHub issue
[akashacms/akasharender#36](https://github.com/akashacms/akasharender/issues/36).

The document was produced after studying `@akashacms/plugins-base`
(`../akashacms-base/index.mjs`) and the akasharender core post-render hook
dispatch (`lib/index.ts`, `Configuration.hookSiteRendered`). It confirms the
issue's reasoning that `<link>`-based oEmbed discovery only needs precomputed
JSON/XML referenced from `<head>` `<link>` tags (no dynamic endpoint), notes
the registry/endpoint discovery path that static sites cannot satisfy, and
lays out a two-piece design (per-page `<head>` injection via Mahabhuta, and
per-page file generation via `onSiteRendered`) gated by a config flag in the
same style as the existing `generateSitemap` flag.

## Analysis

- The oEmbed spec (oembed.com §2.3, §4) supports `<link>`-based discovery
  with static files; the issue's claim is accurate for that mechanism.
- plugins-base already provides the two needed capabilities: `<head>`
  `<link>` injection (`doLinkRelTag`/`doCanonicalURL`) and a post-render
  document-cache scan (`onSiteRendered`, used by the sitemap generator).
- The feature belongs in plugins-base, not akasharender core, alongside the
  existing Open Graph / link-rel metadata handling.
