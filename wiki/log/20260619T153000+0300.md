# 2026-06-19 15:30 - Added Answer Page on Microformats and IndieWeb Markup for Static Blogs

Created a new answer page summarizing the microformats useful for a static
HTML website/blog such as one produced by AkashaCMS, in response to the
microformats/IndieWeb portion of GitHub issue
[akashacms/akasharender#33](https://github.com/akashacms/akasharender/issues/33).
The Fediverse/ActivityPub/Bluesky portions of that catch-all issue are covered
by the two companion answers.

Key points captured:

- Recommend **microformats2** (`h-*`/`p-*`/`u-*`/`dt-*`/`e-*`) over classic v1.
- `h-entry` for posts, `h-card` for authors/people (the author-listing example
  from the question), `h-feed` for index pages.
- `rel="me"` for IndieWeb/RelMeAuth identity (links to the Fediverse/Bluesky
  identity work), `p-category`/`rel="tag"` for tags, and optional Webmention
  reply/like/repost properties.
- Mapped onto AkashaCMS: these are `class`/`rel` annotations on markup the
  partials already produce; the natural injection points are the content/teaser
  partials, the blog-podcast plugin (post structure), and the authors plugin
  (h-card). Cited the current plain-markup partials as the gap.
- Noted verification tooling (microformats2 parsers, IndieWebify.me) and
  marked external specifics NEEDS VERIFICATION.
