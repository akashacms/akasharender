## Analysis

- Read the real package source rather than the compiled `dist` copy in
  `node_modules`. Sources are `src/index.ts` (the `SQ3DataStore` class) and
  `src/finder.ts` (`selectors2where`), TypeScript built with `tsc`, ESM.
- Confirmed the constructor type is `#DB: sqlite3.Database` and the `DB`
  getter returns `sqlite3.Database`; both change to `DatabaseSync` in the
  rewrite. Confirmed all query methods use the
  `new Promise((resolve, reject) => this.#DB.run/all(sql, params, cb))`
  callback pattern that must move to `db.prepare(sql)` + `StatementSync`.
- Confirmed `sqlite3` is a `peerDependency` (not a regular dependency) along
  with `sqlite-regex`; `sqlstring` is the only runtime dependency and stays.
- Found the package ships an existing Mocha test suite (`test/index.mjs`,
  run via `mocha`), which becomes the primary verification for the rewrite
  and must be updated to construct the store with a `DatabaseSync`.
- Recorded that the dropped `AK_PROFILE` SQL profiling is tracked by
  `akashacms/akasharender#192`.
