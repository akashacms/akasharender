## Changes

Created seven new concept documentation files following wiki guidelines:

1. **wiki/concepts/lifecycle-hooks.md** - Comprehensive documentation of the seven lifecycle hooks (beforeSiteRendered, onSiteRendered, onFileAdded, onFileChanged, onFileUnlinked, onFileCacheSetup, onPluginCacheSetup), how Configuration orchestrates sequential execution, and implementation patterns. Sourced from lib/index.ts, lib/Plugin.ts, lib/built-in.ts, and lib/render.ts.

2. **wiki/concepts/custom-elements.md** - Detailed explanation of the Mahabhuta custom element system, the CustomElement wrapper class providing config/akasha/plugin access, implementation patterns, processing flow, and examples from built-in plugin. Sourced from lib/mahafuncs.ts, lib/built-in.ts, and lib/index.ts.

3. **wiki/concepts/database-indexing.md** - Documentation of SQLite indexing strategy covering 17 indexes on Documents table, 3 on TAGGLUE, index types (regular, unique, generated column), and optimization considerations. Sourced from lib/cache/sql/create-table-documents.sql, lib/cache/sql/create-table-tagglue.sql, and lib/cache/schema.ts.

4. **wiki/concepts/command-line-interface.md** - Complete CLI documentation covering 30+ commands organized into categories (rendering, publishing, inspection, tag management, search, development, utility), configuration loading pattern, and error handling. Sourced from lib/cli.ts.

5. **wiki/concepts/cache-schema.md** - Comprehensive schema documentation covering four cache types (Asset, Partial, Layout, Document), base structure, TypeScript/SQLite/Joi three-layer definition pattern, generated columns for JSON extraction, and validation functions. Sourced from lib/cache/schema.ts and create-table-*.sql files.

6. **wiki/concepts/performance-profiling.md** - Documentation of three-level profiling system (document rendering stages using performance.now(), Mahabhuta function execution via FilesystemPerfDataStore, SQL queries via AK_PROFILE environment variable), CLI integration, and configuration. Sourced from lib/render.ts, lib/cli.ts, lib/index.ts, and lib/sqdb.ts.

7. **wiki/concepts/file-watching.md** - Documentation of file system watching using fs.watch API, event detection (add/change/unlink), lifecycle hook invocation, cache synchronization, and selective re-rendering during development. Sourced from lib/cache/watchman.ts and lib/cli.ts.
