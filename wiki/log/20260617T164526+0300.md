# 2026-06-17 16:45 - Implemented node:sqlite Migration and Updated Wiki

Completed the implementation of the migration planned in
[promised-node-sqlite-migration.md](../architecture/promised-node-sqlite-migration.md),
moving AkashaRender's SQLite stack from `sqlite3` + `promised-sqlite3` to the
built-in `node:sqlite` module via the `promised.node.sqlite` async wrapper, and
updated the wiki to reflect the new reality. The source-code changes are outside
the `wiki/` tree and were made under the normal development workflow; this log
records the wiki updates and the implementation outcome.

## Analysis

- The `sq3-kv-data-store` companion package was rewritten to a node:sqlite-only
  v2.0.0 (constructor accepts `DatabaseSync`, queries use `StatementSync`,
  `sqlite3` removed). Its Mocha suite passes (78 tests).
- The `promised.node.sqlite` wrapper gained `undefined`->`null` and
  boolean->`1`/`0` coercion, because `node:sqlite` rejects both at the binding
  boundary. This was the actual cause of the cache insert failures (the prior
  "undefined" theory did not apply; the failing parameter was the boolean
  `$rendersToHTML`). The wrapper's `dist/` is committed so the git dependency
  resolves. Its test suite passes (144 tests, including 4 new boolean tests).
- AkashaRender: `lib/sqdb.ts` reworked (promised.node.sqlite import,
  `allowExtension: true`, sqlite-regex retained, sqlite-vec/sqlite-lembed and
  the embedding registration removed, sqlite3-query-log / AK_PROFILE removed,
  `on('error')` listener removed, `lembedModelName` retained as always-undefined).
  The four `AsyncDatabase` type-only imports were repointed; an unused
  `Database` import and an unused `AsyncDatabase` import were dropped.
- A shared-connection double-close was fixed: the four caches share one global
  `sqdb`, so `BaseCache.close()` no longer closes it and `closeFileCaches()`
  closes it once, guarded by `sqdb.inner.isOpen`.
- Both companion packages are referenced from AkashaRender as git dependencies
  until published to npm.
- Verification: AkashaRender builds clean; the full test suite passes
  (103 + 67 + 117 + 5 + 24 across the five Mocha files, plus the copy-render
  check), with no binding errors and no close-time console noise.

## Changes

- Updated `wiki/summaries/lib/sqdb.ts.md` to describe the node:sqlite /
  promised.node.sqlite setup, `allowExtension: true`, the removal of
  sqlite-vec/sqlite-lembed and SQL profiling, the dormant `lembedModelName`,
  and the removed error listener.
- Updated `wiki/concepts/sqlite-database.md`: definition, initialization
  snippet, extensions section, profiling section, AsyncDatabase API, error
  handling, key parameters, when-to-use, pitfalls, and related pages — all to
  reflect node:sqlite, the wrapper's type coercion, the single-close behavior,
  and the removed features.
- Updated `wiki/concepts/key-value-store.md`: node:sqlite-backed v2.0.0
  package, `sqdb.inner` is now a `DatabaseSync`, corrected the usage example to
  the real API (`put`/`get`), documented the full public API, and updated the
  dependency note.
